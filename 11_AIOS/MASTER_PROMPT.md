# MASTER PROMPT — প্রতিটা নতুন AI সেশনের শুরুতে এটা পেস্ট করো

তুমি "relife-clinic-os" নামের একটা লাইভ Telegram ক্লিনিক-ম্যানেজমেন্ট
বটের উপর কাজ করছ। এই প্রজেক্টে তুমি একা নও — একাধিক AI (Claude/ChatGPT/
Gemini/DeepSeek/Grok ইত্যাদি) একসাথে বিভিন্ন মডিউলে কাজ করছে, তাই নিচের
নিয়মগুলো বাধ্যতামূলকভাবে মানতে হবে।

## ধাপ ০ — রেজিস্ট্রেশন (সবার আগে, বাধ্যতামূলক)
কাজ শুরুর আগে নিজেকে একটা ফিক্সড ID দাও (11_AIOS/AI_REGISTRY.md-এর
তালিকা থেকে, যেমন Claude-3, ChatGPT-2)। ব্যবহারকারী তোমাকে ID বলে
না দিলে নিজে থেকে জিজ্ঞাসা করো: "আমি কোন ID ব্যবহার করব?"
এই ID-ই তোমার প্রতিটা HANDOVER.md এন্ট্রিতে এবং TASK_QUEUE.md-এ
ব্যবহার হবে — কোনো আসল নাম/Gmail নয়।

## শুরু করার আগে অবশ্যই পড়ো
1. 11_AIOS/AI_CONSTITUTION.md — মূল নিয়মকানুন।
2. 11_AIOS/AI_REGISTRY.md — কোন ID ইতিমধ্যে ব্যবহৃত/বরাদ্দ।
3. 12_Handover/HANDOVER.md — আগের AI কী করে গেছে।
4. 13_AI_Tasks/TASK_QUEUE.md — কোন কাজ Pending/In-Progress/Done।

## তোমার কাজের সীমানা
- ব্যবহারকারী শুধু তোমাকে যে নির্দিষ্ট মডিউল/ফাইল বরাদ্দ করবে, শুধু সেটাই
  ছোঁবে। অন্য মডিউলে হাত দেবে না, এমনকি "উন্নতি" মনে হলেও না।
- 03_Bot-এর কোনো ফাইল পুরো ওভাররাইট নয় — শুধু sandbox-টেস্টেড
  exact-string patch script (patchN.py)।
- config.py-এর কন্টেন্ট অনুমান কোরো না।

## আউটপুট ফরম্যাট
- ব্যবহারকারী Termux (Android) ব্যবহার করে — টার্মিনালে সরাসরি
  পেস্টযোগ্য কমান্ড/heredoc/script দাও, আংশিক কোড না।
- অপ্রয়োজনীয় ব্যাখ্যা ছাড়া শুধু প্রয়োজনীয় কমান্ড দাও।
